# Moe's Vocab Specification

## SM-2 即時計算・セッション管理アーキテクチャ

### 概要
深夜・早朝の学習（日またぎ）による日付のズレを手動で修正する運用をゼロにするため、バッチ処理ではなくフロントエンド（ブラウザ）での**「時間ベースの完全即時計算モデル」**を採用しています。
基準時刻はユーザー端末（ブラウザ）の時刻ではなく、スクリプト（サーバー）が `words.json` を生成した時刻（`meta.version` / UNIXタイムスタンプ）を使用し、DBの時刻とも整合性を保ちます。

### 1. 時間とセッションの定義
*   **基準時刻 (Server Time)**: `words.json` の `meta.version`（UNIXタイムスタンプ・秒）を絶対的な現在時刻として扱います。
*   **セッション (Session ID)**: 同様に `meta.version` を「今回の学習セッションの一意なID」として扱います。

### 2. データ構造 (Supabase: `progress_sync` テーブル)
*   `last_session_version` (bigint): その単語が最後に評価された時のセッションID (`meta.version`)。
*   `next_review` (timestamptz / ISO8601): 次回復習可能になる絶対時刻。

### 3. フロントエンドの動作仕様 (`index.html`)
学習ボタン（✅ / ❌）押下時、以下のフローで処理を行います。

1.  **セッション判定**:
    *   `word.last_session_version === wordsData.meta.version` なら「**同じセッションの解き直し（もう一回）**」と判定。
    *   異なっていれば「**新しいセッションの初回学習**」と判定。
2.  **SM-2計算 & 保護ロジック**:
    *   **初回学習**: SM-2アルゴリズムを実行し、`ease_factor`, `interval_days`, `repetitions` を更新。
    *   **解き直し**: 前回が✅で今回が❌に悪化した場合**のみ**、ペナルティとしてSM-2をリセット（quality=0）して再計算。それ以外は計算をスキップし、前回の評価を保持（最悪評価の保護）。
3.  **次回復習日時の算出**:
    *   `next_review` = `(meta.version * 1000) + (interval_days * 24 * 60 * 60 * 1000)` にて絶対時刻を算出。
4.  **保存**:
    *   `last_session_version`, `next_review`, `ease_factor`, `interval_days`, `repetitions`, `status` を即時 `upsert` する。

## 単語選択ロジック (2026-03-25 更新)

### 基本パラメータ
- **1日の最大単語数 (DAILY_LIMIT)**: 50
- **新規単語の最低保証 (MIN_NEW)**: 10
- **復習枠の最大数**: 40 (DAILY_LIMIT - MIN_NEW)

### 分類ルール
学習済みの全単語を以下の3つのバケットに分類します。判定基準は `next_review`（ISO8601日時）と現在時刻の比較です。

1.  **red (最優先復習)**: `next_review <= 現在時刻` かつ、直近の評価（`status`）が `red` である単語。
2.  **due (通常の復習)**: `next_review <= 現在時刻` かつ、直近の評価（`status`）が `green` である単語。
3.  **new (新規)**: まだ一度も学習していない（`next_review` が未設定の）単語。
    - 優先順位: Part降順 (3->2->1) -> Section昇順 -> ID昇順

### 選択アルゴリズム
合計50語を選定する際、以下の優先順位で枠を埋めます。

1.  **復習枠 (最大40語)**:
    - まず **red** バケットから抽出。
    - 余った枠で **due** バケットから抽出。
    - (合計で最大40語まで)
2.  **新規枠 (最低10語 + α)**:
    - **MIN_NEW (10語)** は、復習枠の状態に関わらず必ず含める。
    - 復習枠（red + due）が40語に満たなかった場合、余った枠もすべて **new** で埋め、合計50語にする。
